Privacy Policy

Last updated: 10 May 2026

This Privacy Policy describes how the internal automation tool ("the Tool") operated by Kerem Yilmaz Digital Marketing ("we", "us") handles information accessed through the Meta Marketing API and the Google Ads API. The Tool is for internal use only and is not made available to the public, to clients, or to any third party.

1. Scope

The Tool is operated solely by Kerem Yilmaz, a freelance digital marketing consultant based in Turkey, for the purpose of automating reporting, monitoring, and rule-based optimization tasks across advertising accounts that Kerem Yilmaz Digital Marketing has been authorized to manage by their respective owners.

2. Information We Access

The Tool accesses the following categories of information through authorized API connections:

  • Advertising performance metrics (impressions, clicks, cost, conversions, CTR, CPM, CPC, ROAS) for managed campaigns
  • Campaign, ad set, and ad configuration data (names, status, budgets, schedules, targeting parameters)
  • Conversion and pixel event data for managed pixels
  • Ad account metadata required to identify and address the correct managed account

The Tool does not access, collect, or process personally identifiable information (PII) about end users (the consumers who view, click on, or convert from our clients' ads). Any user-level data exposed by the platforms is aggregated before any human review.

3. How Information Is Used

Information accessed through the API is used exclusively for:

  • Generating performance reports for the account owner
  • Sending monitoring alerts to the operator (e.g. via Telegram) when performance metrics deviate from defined thresholds
  • Applying pre-approved, rule-based optimizations such as pausing under-performing campaigns or making bounded budget adjustments

Information accessed for one client's account is never used in connection with another client, and is never disclosed to any third party.

4. Data Storage and Retention

API credentials (developer tokens, OAuth tokens, app secrets) are stored in environment variables on a private server with restricted access; they are never written to logs, never committed to version control, and never shared.

Raw API responses are not persisted. Aggregated report outputs (PDF, Markdown summaries) are retained for up to 90 days for audit and historical comparison purposes, then automatically deleted. Operational log files are rotated weekly and removed after 30 days.

5. Data Sharing

We do not sell, license, or otherwise share data accessed through the API with any third party, with the following narrow exceptions:

  • The respective platform (Meta or Google) for the purpose of making the authorized API call itself.
  • The owner of the managed account, who receives reports derived from data accessed in their own account.
  • Telegram, used solely as the delivery channel for rendered summary messages and alerts addressed to the operator. No raw credentials or unaggregated user-level data are transmitted via Telegram.

6. Security

Access to the server running the Tool is restricted to the operator (Kerem Yilmaz) and protected by SSH key authentication. The server receives security updates regularly. There is no public-facing service exposed by the Tool.

7. Account Owner Rights

Owners of managed advertising accounts may at any time:

  • Revoke the Tool's access to their account through their Meta Business Manager or Google Ads account permissions interface.
  • Request deletion of any retained report outputs that pertain to their account.
  • Request a description of what data has been accessed by the Tool in connection with their account.

Such requests can be sent to the contact address below and will be honored within 30 days.

8. Compliance

The Tool is operated in compliance with the Meta Platform Terms, the Meta Developer Policies, the Google Ads API Terms and Conditions, and the Google API Services User Data Policy, including the Limited Use requirements where applicable.

9. Changes to This Policy

We may update this policy from time to time as the scope of the Tool evolves. The "Last updated" date at the top of this document reflects the most recent revision.
